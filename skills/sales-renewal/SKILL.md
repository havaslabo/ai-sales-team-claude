# 契約更新・アップセル支援スキル

既存顧客の契約更新・アップセル・カスタマーサクセス商談を支援するスキルです。

## 呼び出し方

```
/sales renewal <company_name>
```

## 入力

- 企業名
- 現在の契約内容（プラン・金額・契約期間）
- 利用状況（活用機能・利用頻度・担当者数等）
- 更新期限

## 出力

`RENEWAL-PROPOSAL-[会社名].md`

---

## 目的

既存顧客の更新率（リテンション）向上とアップセルは新規獲得より費用対効果が高い。日本の大企業は担当者の人事異動で関係がリセットされるリスクがあるため、更新前の関係強化と新担当者への引き継ぎが重要。

- 新規顧客獲得コストは既存顧客維持の5〜7倍とされる
- 日本企業は4月・10月の人事異動で担当者が変わるケースが多い
- 更新期限の3〜6ヶ月前から動き始めることが成功のカギ

---

## 生成する内容

### 1. 更新商談準備ブリーフ

社内向けの準備資料。商談前に状況を整理する。

- 現在の利用状況のサマリー（活用できている機能・使われていない機能）
- 導入からの成果・実績（定量的に）
- 更新リスク要因の特定
  - 低活用（ログイン頻度・機能利用率が低い）
  - 担当者異動（特に4月・10月）
  - 予算削減（期末・新年度予算）
  - 競合への乗り換え検討
- 推奨アクション（優先順位付き）

### 2. 更新提案書

顧客提出用の正式提案書。

- 「導入から〇ヶ月間のご利用実績レポート」
- 達成できた効果（KPI）を定量的に提示
- 次年度の活用提案（追加機能・拡張プラン）
- 更新価格・早期更新特典（例: 1ヶ月前更新で5%割引）

### 3. アップセル提案

現在の契約を超えた価値提供の提案。

- 現在未利用の機能・プランのご提案
- 追加導入による追加効果の見込み（ROI試算）
- 他部署・グループ会社への横展開提案
- 段階的な拡張ロードマップ

### 4. 担当者異動対応テンプレート（4月・10月の人事異動シーズン用）

| テンプレート | 用途 |
|------------|------|
| 旧担当者への御礼メール | 長期間の関係に感謝し、引き継ぎをサポート |
| 新担当者への引き継ぎ・挨拶メール | 関係ゼロから始める初回コンタクト |
| 「改めてご説明の機会」依頼メール | 関係リセット時の打ち合わせ設定依頼 |

---

## 重要なルール

1. **更新期限の3〜6ヶ月前から活動を開始する**
   - 1ヶ月前では遅い。競合への乗り換え検討が始まっている可能性がある。

2. **利用データに基づいた具体的な成果を示す**
   - 「〜に役立ちます」ではなく「〜が〇%改善されました」という実績ベースで話す。

3. **アップセルは「押し売り」ではなく「さらなる価値提供」として提案する**
   - 顧客の課題・目標に基づいて提案する。価格の話は価値提示の後に行う。

4. **担当者異動は必ずフォローする（日本では4月・10月が多い）**
   - 新担当者への引き継ぎを怠ると、関係がゼロにリセットされる。
   - 異動情報はLinkedIn・会社HP・名刺交換等で早期に把握する。

---

## 実行フロー

1. 企業名・契約情報・利用状況・更新期限をヒアリング
2. 更新リスクを評価（高・中・低）
3. 更新商談準備ブリーフを生成（社内向け）
4. 更新提案書を生成（顧客向け）
5. アップセル提案を生成（顧客向け）
6. 担当者異動の有無を確認し、必要に応じてメールテンプレートを生成
7. `RENEWAL-PROPOSAL-[会社名].md` として出力

---

## 注意事項

- 利用状況データは顧客から入手するか、社内CRM・ツールから取得すること。
- アップセル提案は顧客の予算サイクル（日本企業は4月始まりが多い）を考慮する。
- 早期更新特典を提供する場合は、社内承認を先に取ること。
- 更新提案書は稟議資料としても使われるため、承認者（決裁者）が読んでも分かる内容にする。
