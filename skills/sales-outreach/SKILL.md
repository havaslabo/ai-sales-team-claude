# アウトリーチシーケンスジェネレーター（日本BtoB向け）

`/sales outreach <prospect>` のアウトリーチエンジンです。完全にパーソナライズされた、すぐに送信できるメールシーケンスを生成します。すべてのメールは日本のBtoB営業文化に基づき、実際のパーソナライゼーションデータで調整されています。汎用テンプレートは使用しません。このスキルは単独で呼び出すか、`/sales prospect` 内で **sales-strategy** サブエージェントとして使用します。

## このスキルが呼び出されるタイミング

- **単独使用:** ユーザーが `/sales outreach <prospect>` を実行します。`<prospect>` にはURL、会社名、または既存の分析への参照を指定できます。パーソナライゼーションリサーチを実施し、OUTREACH-SEQUENCE.md を出力します。
- **サブエージェントとして:** sales-prospect オーケストレーターがこのスキルを sales-strategy サブエージェントとして起動します。事前に取得したページコンテンツを含むディスカバリーブリーフィングを受け取ります。それを活用して重複したリサーチをスキップします。Outreach Readiness Score（0〜100）とフルシーケンスを返します。

---

## チャネル優先戦略（日本BtoB向け）

**重要なルール:** 日本のBtoB営業では、関係構築を最優先にします。最初の接触で提案を押しつけず、まず信頼と接点を作ることが受注への近道です。

### チャネル優先順位

1. **紹介・ウォームイントロ（最優先）** — 共通の知人・取引先・業界人脈を通じた紹介が最も効果的。まず紹介を探すことから始める。
2. **メール** — ビジネスメール文化が根付いており、日本語ビジネス文体で丁寧に送る。
3. **電話（テレアポ）** — 決裁者への直接フォローアップとして有効。ただし事前にメールで接点を作ってから。
4. **業界展示会・セミナー** — 対面での接点づくりに有効。名刺交換後のフォローアップに繋げる。
5. **LinkedIn（任意・補助的）** — 日本での普及率は限定的。メインチャネルとしては使用しない。補助的な情報収集や接点確認に活用する程度とする。

### コールドメールの位置づけ

コールドメール（紹介なしの初回接触）は最終手段です。まず以下を確認してください：
- 社内や既存顧客に紹介できる人物がいないか
- 同じ業界団体・セミナーに参加していないか
- 展示会やイベントで接点を作れないか

紹介が見つからない場合に初めてコールドアウトリーチを実施します。

## フェーズ1: パーソナライゼーションリサーチ（メール執筆前に必ず実施）

**重要なルール:** パーソナライゼーションリサーチを完了する前に、1通もメールを書いてはいけません。汎用的なアウトリーチは、何もしないよりも悪い結果をもたらします。すべてのメールには、見込み客または担当者に関する具体的で検証可能な参照を少なくとも1つ含める必要があります。

### 1.0 紹介ルートの事前確認（最優先ステップ）

アウトリーチを始める前に、まず以下を確認します:

```
紹介ルート確認チェックリスト
============================
- 自社の既存顧客・パートナーで、ターゲット企業と取引があるか？
- 業界団体・商工会議所の共通メンバーはいるか？
- 同じセミナー・展示会に出展・参加しているか？
- 採用情報（Wantedly等）から人脈が見つかるか？
- 対象者の登壇・寄稿情報から接点が作れるか？

紹介ルートあり → outreach-warm.md のプロセスを使用
紹介ルートなし → 以下のコールドアウトリーチリサーチへ進む
```

### 1.1 会社トリガーリサーチ

`WebSearch` を使用して最近の会社トリガーを調べます。これらは自然でタイムリーなアウトリーチの理由を生み出します。

**実行する検索クエリ:**
```
"[会社名] 資金調達"
"[会社名] 新サービス"
"[会社名] 採用"
"[会社名] 提携"
"[会社名] 拡大"
"[会社名] ニュース"
"[会社名] 代表 インタビュー"
```

**参照するデータソース（日本向け）:**
- 企業公式サイト、IR情報
- Wantedly（採用・企業文化）
- INITIAL（entrepedia）、SPEEDA（資金調達・財務情報）
- 日本政策投資銀行データ、業界紙・業界誌
- 業界団体サイト、プレスリリース配信サービス（PR TIMES等）

**トリガーイベントのカテゴリー:**

| トリガーカテゴリー | 検出シグナル | アウトリーチの切り口 | 鮮度要件 |
|-----------------|-------------------|----------------|----------------------|
| **最近の資金調達** | プレスリリース、INITIAL、ニュース記事 | 「資金調達のニュースを拝見しました。急成長時には[あなたが解決する問題]が生じることが多く、お役に立てるかと思いご連絡いたしました」 | 直近6ヶ月以内 |
| **製品・サービスローンチ** | ブログ記事、プレスリリース | 「[サービス]のリリースを拝見しました。同様のフェーズの企業様のお手伝いをしてきた経験があります」 | 直近3ヶ月以内 |
| **採用拡大** | Wantedly、求人サイト、新部署設立 | 「組織が拡大されていることを拝見しました。このような時期に[課題]が生じるケースが多くございます」 | 直近3ヶ月以内 |
| **リーダーシップ変更** | プレスリリース、企業サイト | 「ご就任のニュースを拝見しました。新しい体制でのご取り組みをぜひ伺えればと思っています」 | 直近3ヶ月以内 |
| **事業拡大** | 新拠点設立、新市場参入 | 「[地域]への展開を拝見しました。類似した事業展開を支援してきた実績がございます」 | 直近6ヶ月以内 |
| **受賞・表彰** | 業界賞、メディア掲載 | 「[受賞]のニュースを拝見しました。[エリア]での取り組みが評価された結果と存じます」 | 直近6ヶ月以内 |

**トリガーの品質評価:**
- **ホットトリガー（即座に使用）:** 直近30日以内に発生。あなたのソリューションに直接関連している。
- **ウォームトリガー（1週間以内に使用）:** 直近90日以内に発生。あなたのソリューションエリアに関連している。
- **クールトリガー（背景情報として使用）:** 3〜6ヶ月前に発生。背景情報にはなるが、緊急性はない。

## フェーズ2: アウトリーチフレームワークの選択

### 2.1 4つのアウトリーチフレームワーク

パーソナライゼーションリサーチの結果に基づき、最適なフレームワークを選択します:

#### フレームワーク1: 観察 → 課題提示 → 相談打診

**構成:**
```
「[相手または会社についての具体的で検証可能な観察]を拝見いたしました。」
「[類似の状況における課題や背景を丁寧に示すブリッジ]。」
「[ソフトな相談打診：まず情報交換・意見交換の場をお願いする]。」
```

**使用場面:**
- 強力なパーソナライゼーションアンカーが見つかった場合（記事、登壇、新サービス等）
- 観察内容とあなたの提案の間の関連性が自然な場合

#### フレームワーク2: 課題提示 → 事例紹介 → 相談打診

**構成:**
```
「[具体的な業界課題の提示 — 押しつけにならず、観察として丁寧に]。」
「[類似企業・類似案件での取り組み事例（数値より課題解決の文脈を重視）]。」
「[相談打診：まず状況をお聞かせいただきたいというスタンスで]。」
```

**使用場面:**
- 相手の状況に関連した事例・実績がある場合
- 業界・セグメントで広く経験されている問題の場合

#### フレームワーク3: トリガーイベント活用

**構成:**
```
「[特定の最近のトリガーイベントを日付・詳細とともに参照]。」
「[このイベントがあなたの提案との関連性を生み出す理由を丁寧に説明]。」
「[相談打診：タイムリーな接点を自然につなげる]。」
```

**使用場面:**
- ホットまたはウォームな会社トリガーが見つかった場合（資金調達、ローンチ、採用、拡大）
- トリガーがあなたのソリューションのニーズを直接生み出している場合

#### フレームワーク4: 紹介・共通接点の活用

**構成:**
```
「[共通のコネクション、業界団体、イベントへの参加を参照]。」
「[その共通点からビジネスの関連性へのブリッジ]。」
「[相談打診：対等な立場での意見交換として]。」
```

**使用場面:**
- 本物の共通コネクション、共有コミュニティ、または共通の背景がある場合
- シニアな意思決定者との信頼を素早く構築するのに最適

### 2.2 フレームワーク選択のロジック

この決定木を使用します:

```
紹介や共通の接点はありますか？
  YES → フレームワーク4（紹介・共通接点）を使用 → outreach-warm.mdも参照
  NO ↓

ホットな会社トリガーイベントはありますか？
  YES → フレームワーク3（トリガーイベント）を使用
  NO ↓

強力なパーソナライゼーションアンカー（記事、登壇、新サービス）はありますか？
  YES → フレームワーク1（観察 → 課題提示 → 相談打診）を使用
  NO ↓

関連する事例・実績はありますか？
  YES → フレームワーク2（課題提示 → 事例紹介 → 相談打診）を使用
  NO → 業界レベルの観察でフレームワーク1を使用（コールドメールは最終手段）
```

## フェーズ3: 3通のメールシーケンス（日本向け）

**日本BtoB向け短縮シーケンス:** 5通は「しつこい」と受け取られるリスクがあります。3通で完結させ、電話フォローアップと組み合わせます。

### メール執筆ルール（すべてのメールに適用）

**件名のルール:**
- 「株式会社〇〇 山田様へ / [用件]」の形式を基本とする
- 社名・担当者名を明記する
- 感嘆符を使わない
- 全大文字を使わない
- スパムトリガーワードを使用しない（「無料」「保証」「今すぐ」「期間限定」等）

**書き出しのルール:**
- 「突然のご連絡、失礼いたします」から始める（コールドの場合）
- 「[MUTUAL_CONNECTION] 様よりご紹介いただきました」（紹介の場合）
- 「先日は[イベント名]にてご縁をいただきありがとうございました」（対面接点がある場合）
- 自己紹介を簡潔に行う（会社名・氏名・役職）

**本文のルール:**
- メール1〜2: 合計200〜300字程度（日本語）
- メール3: 合計150字程度
- 短い段落（各1〜3文）
- ビジネス敬語を使う
- 「P.S.」は使用しない
- 締めは「よろしくお願いいたします」「以上」「何卒よろしくお願い申し上げます」などの日本語ビジネス文体
- 最初の接触で価格や契約の話は一切しない

**CTAのルール:**
- 1通のメールにつき1つのお願い
- 「ご都合のよろしい際に、一度お話しする機会をいただければ幸いです」
- 「まずは30分ほど、情報交換の場をいただけますでしょうか」
- 強引なデモやプレゼンの要求はしない
- 断りやすい言葉を必ず添える（「ご多用の折、誠に恐れ入りますが」等）

**トーンのルール:**
- 売り込みではなく、課題解決・業務改善の観点から
- まず相手の状況を理解したいというスタンス
- ROI数値の羅列より、具体的な課題感への共感を優先
- 丁寧・謙虚だが、専門性を示す
- 絵文字を使わない

### メール1 — 初回接触（1日目）

**目的:** 礼儀正しく接点を作る。まず信頼できる存在として認識してもらう。

**構成:**
```
件名: 株式会社[COMPANY_NAME] [LAST_NAME]様へ / [用件の概要]

[書き出し: 突然のご連絡の失礼をお詫び — 1文]

[自己紹介: 会社名・氏名・役職 — 1〜2文]

[パーソナライゼーション: 先方の取り組み・ニュースへの言及 — 1〜2文]

[接触理由: 類似業界での課題解決の経験を自然に提示 — 1〜2文]

[相談打診: 柔らかいお願い — 1文]

[締め: よろしくお願いいたします / 氏名・会社名・連絡先]
```

### メール2 — 価値提供（3日目）

**目的:** 何も求めずに純粋な価値を提供する。有益な情報・事例を届けて信頼を築く。

**構成:**
```
件名: Re: [PREVIOUS_SUBJECT] または [RESOURCE_TITLE]について — [YOUR_COMPANY]

[前回連絡への簡単な言及 — 1文、罪悪感を抱かせない]

[有益なリソース・業界情報・事例の紹介 — 2〜3文]
※先方の業界・課題に本当に関連するものだけ送る

[なぜ先方に関連すると思ったかの説明 — 1文]

[締め: ご参考になれば幸いです / 氏名]
```

**重要な原則:** このメールは与えるものであり、求めるものではありません。返信を促すCTAは不要です。

### メール3 — クロージング（7日目）

**目的:** 丁重に締めくくりつつ、将来の接触を可能にする。

**構成:**
```
件名: Re: [PREVIOUS_SUBJECT]

[これ以上ご連絡しない旨を丁寧に伝える — 1文]

[最後に一度だけ価値提案を簡潔に再提示 — 1〜2文]

[将来的な接点の可能性を残す — 1文]

[締め: 引き続きよろしくお願いいたします / 氏名・連絡先]
```

## フェーズ4: 電話フォローアップ（LinkedIn の代替）

日本のBtoB営業では、LinkedInよりも電話フォローアップが効果的です。

### 4.1 電話フォローアップのタイミング

| 日程 | チャネル | アクション |
|------|---------|---------|
| **1日目** | メール | メール1（初回接触）を送信 |
| **3日目** | メール | メール2（価値提供）を送信 |
| **5日目** | 電話 | 「先日メールをお送りした件で」と切り出すテレアポ |
| **7日目** | メール | メール3（クロージング）を送信 |
| **10日目** | 電話 | 最終フォローアップ電話（必要な場合） |

### 4.2 テレアポの切り出しトーク

```
「お忙しいところ失礼いたします。
[YOUR_COMPANY] の [YOUR_NAME] と申します。
先日、[用件の概要] につきましてメールをお送りしておりましたが、
ご確認いただけましたでしょうか？

もし差し支えなければ、[具体的な相談内容] について
5分ほどお時間をいただけますでしょうか。」
```

**電話のルール:**
- 午前10時〜12時、午後2時〜4時が比較的つながりやすい
- 「突然のお電話で失礼いたします」から始める
- 事前に送ったメールに言及して接点を作る
- 担当者不在の場合は折り返しをお願いするか、かけ直す時間を確認

### 4.3 LinkedIn（補助的・任意）

LinkedInは以下の限定的な用途にのみ活用します：
- 相手の役職・部署の確認（人事異動・組織情報）
- 業界キーパーソンの情報収集
- 接続リクエストは電話またはメールで接点ができた後に送る

## フェーズ5: 送信のベストプラクティス（日本向け）

### 5.1 メール到達率

| プラクティス | 推奨事項 |
|----------|---------------|
| **送信量** | 受信トレイあたり1日5〜10件から始め、徐々に増やす |
| **ドメイン設定** | SPF、DKIM、DMARCが設定されていることを確認する |
| **配信停止** | 配信停止リクエストは即座に対応し、すべてのシーケンスから除外すること |
| **バウンス処理** | バウンスしたアドレスは即座に削除する |

### 5.2 送信タイミング（日本BtoB向け）

| 対象 | 最適な曜日 | 最適な時間帯 | 避けるべき時間 |
|----------|-----------|-----------|-------|
| **経営者・役員クラス** | 火・水・木 | 午前8〜9時または午後6〜7時 | 月曜午前、金曜午後、月初月末 |
| **部長・マネージャー** | 火・水・木 | 午前9〜11時 | 月曜午前、金曜午後 |
| **担当者クラス** | 火・水・木 | 午前10時〜午後12時 | 月曜、金曜 |

**日本固有の注意事項:**
- 年度末（3月）・年度初め（4月）は予算決裁のタイミング
- お盆（8月中旬）・年末年始（12月下旬〜1月上旬）は避ける
- 月初・月末は月次業務で忙しいため、月中を狙う

### 5.3 フォローアップのルール

| シナリオ | アクション |
|----------|--------|
| **3通すべてに返信なし** | 30日待ち、別のアングルまたはトリガーイベントで再アプローチする |
| **不在通知の返信** | 帰社日をメモし、帰社翌日にメール1を再送信する |
| **「結構です」の返信** | 丁寧に返信する。シーケンスから外す。他にご担当者をご紹介いただけるか確認する。 |
| **「今は時期でない」の返信** | いつが良いか確認する。リマインダーを設定する。その間に価値あるリソースを送る。 |
| **ポジティブな返信** | 1営業日以内に返信する。具体的なミーティング日程を2〜3つ提案する。 |

## エラーハンドリング

- パーソナライゼーションデータが見つからない場合は制限事項を明記し、業界レベルのパーソナライゼーション（最も弱いアプローチ）を使用する
- コンタクトのオンライン情報が見つからない場合は、会社レベルのパーソナライゼーションのみで進める
- メール3向けの事例や証拠ポイントがない場合は、業界ベンチマークまたはサードパーティのデータを使用する
- 見込み客企業のオンラインプレゼンスが非常に少ない場合は、業界トリガーと一般的な課題感に焦点を当てる
- パーソナライゼーションの質に関わらず、常に3通のメールシーケンスを完全に生成する。ただし、弱いパーソナライゼーションに依存している場合は明記する
- サブエージェントとして実行していて時間が限られている場合は、メール2〜3の完成度よりもメール1の品質を優先する

## クロススキル統合

- `COMPANY-RESEARCH.md` が存在する場合は、パーソナライゼーションとコンテキストに会社データを使用する
- `DECISION-MAKERS.md` が存在する場合は、コンタクトプロフィールとパーソナライゼーションアンカーを使用する
- `LEAD-QUALIFICATION.md` が存在する場合は、メッセージングの問題点と購買シグナルを使用する
- `COMPETITIVE-INTEL.md` が存在する場合は、差別化アングルの競合ポジショニングを使用する
- 返信を得た後のミーティング準備には `/sales prep`、ミーティング後のシーケンスには `/sales followup`、より詳細な反論対処には `/sales objections` を提案する
