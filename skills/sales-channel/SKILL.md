# 代理店・パートナー営業支援スキル

代理店・パートナー・SIer経由の間接販売（チャネル営業）を支援します。日本のBtoB市場では、エンドユーザーへの直販より、SIer・代理店・商社・コンサル経由の間接販売が主流です。パートナーとの協業提案・パートナー向け営業資料・エンドユーザー向け合同提案書を生成します。

## 呼び出し方法

```
/sales channel <partner_company>
```

`<partner_company>` はパートナー企業名です。エンドユーザー企業名は任意で指定できます。

## ステップ1：チャネル情報の収集

以下の情報を収集します。ユーザーから提供されない場合は、各項目を明示的に確認してください。

### 必須インプット

1. **パートナー企業名**：正式名称・略称
2. **パートナーの業種・種別**：以下から選択（複数可）
   - SIer（システムインテグレーター）
   - VAR（付加価値再販業者）
   - OEM・ホワイトラベル
   - コンサルティングファーム
   - 商社・流通
3. **パートナーの強み・専門領域**：得意業界、保有顧客層、技術力、ブランド力
4. **自社製品・サービスの概要**：提供するソリューション・価値

### 任意インプット

- **エンドユーザー企業名**：特定の案件がある場合
- **エンドユーザーの業種・規模**
- **想定する協業モデル**（例：共同受注・紹介手数料・OEM供給等）
- **既存のパートナー関係**：新規開拓か既存パートナーの深耕か
- **価格帯・マージン率の目安**

## ステップ2：対応するチャネル種別

### SIer（システムインテグレーター）
NTTデータ・富士通・NEC・アクセンチュア・IBMなど大手から中堅SIerまで対応。

- **特徴**：顧客との長期関係・システム全体の設計・導入・保守を担う
- **協業ポイント**：SIerが受注した案件に自社製品を組み込んでもらう。技術支援・検証環境の提供が重要
- **注意点**：SIerは顧客との直接関係を重視するため、パートナーを介さず顧客にアプローチしない

### VAR（付加価値再販業者）
自社製品を他のソリューションと組み合わせて提供するパートナー。

- **特徴**：特定業界・用途に特化した付加価値を提供
- **協業ポイント**：マージン設計・サポート体制・トレーニングプログラムの整備が重要
- **注意点**：VARの顧客リストは機密。プッシュではなくプル型の支援が有効

### OEM・ホワイトラベル
パートナーブランドで自社製品・技術を提供。

- **特徴**：自社ブランドを前面に出さず、パートナーのブランド強化に貢献
- **協業ポイント**：ブランド非表示での提供条件・ライセンス体系・カスタマイズ対応範囲の明確化
- **注意点**：エンドユーザーに自社の存在が見えない場合があるため、品質保証・サポートの取り決めを明確にする

### コンサルティングファーム
戦略・業務改善コンサルと一緒に提案するモデル。

- **特徴**：C層・経営層へのアクセスが強い。業務課題起点で提案できる
- **協業ポイント**：コンサルの提言の中に自社ソリューションを組み込む。ROI・業務改善効果の数値化が重要
- **注意点**：コンサルは中立性を重視するため、押し売りにならない提案が必要

### 商社・流通
製造業・建設業・食品業など特定業界への商社経由アプローチ。

- **特徴**：業界ネットワークと既存取引関係が強み
- **協業ポイント**：商社の既存顧客基盤を活用した横展開。シンプルな価格体系と手厚いサポートが重要
- **注意点**：商社は多くの製品・サービスを扱うため、自社製品の優先度を高く保つ働きかけが必要

## ステップ3：生成する内容

### 1. パートナー向け提案書（協業のメリット）

パートナー社内の意思決定者に向けた協業提案書。

```markdown
## [パートナー名]との協業提案

### なぜ[パートナー名]と協業するのか
[パートナーの強みと自社ソリューションの補完関係]

### パートナーにとってのビジネスメリット
1. **収益機会の拡大**：[マージン率・想定売上規模]
2. **顧客提案力の強化**：[新たな提案領域・差別化ポイント]
3. **顧客基盤の拡大**：[新規顧客獲得の可能性]

### 自社のサポート体制
- **技術支援**：[技術トレーニング・検証環境・SE支援]
- **営業支援**：[共同商談・提案書支援・マーケティング素材提供]
- **優先サポート**：[パートナー専用窓口・SLA・エスカレーション体制]

### 協業モデルの概要
| 項目 | 内容 |
|------|------|
| 契約形態 | [代理店契約・OEM契約・紹介契約等] |
| マージン率 | [XX%（税別）] |
| 最低コミット | [有無・条件] |
| 更新条件 | [年次更新・実績連動等] |
```

### 2. パートナー経由のエンドユーザー向け合同提案書

「[パートナー名]と[自社名]の協業ソリューション」という形式での提案書。稟議を通りやすい形式を採用します。

```markdown
## [パートナー名] × [自社名] 協業ソリューション提案書

**提出先：** [エンドユーザー企業名] 御中
**提出者：** [パートナー名] [担当者名] / [自社名] [担当者名]

### ソリューション概要
[パートナーの強み] × [自社製品の機能] を組み合わせることで、
[エンドユーザーの課題] を解決します。

### なぜ[パートナー名]との組み合わせが最適か
- [パートナーの実績・信頼性]
- [自社製品との相乗効果]
- [稟議を後押しするポイント：保証・サポート・実績]

### 導入・運用体制
- **導入支援**：[パートナー担当] が一括窓口
- **技術サポート**：[自社名] が[パートナー名]を通じてバックアップ
- **契約・請求**：[パートナー名] と一元管理
```

### 3. 共同商談の役割分担アジェンダ

パートナーと自社が共同でエンドユーザー商談に臨む際の役割分担を明示します。

```markdown
## 共同商談 進行アジェンダ

**日時：** YYYY年MM月DD日
**商談先：** [エンドユーザー企業名]

| 時間 | 内容 | 担当 |
|------|------|------|
| 00:00〜 | 挨拶・自己紹介 | パートナー主導 |
| 00:05〜 | 顧客の課題・現状確認 | パートナー主導（自社はヒアリングサポート） |
| 00:20〜 | ソリューション概要説明 | 共同（パートナー：業務視点 / 自社：技術・製品視点） |
| 00:40〜 | デモ・事例紹介 | 自社主導（パートナーが文脈を補足） |
| 00:55〜 | 質疑応答・次のステップ確認 | パートナー主導（自社はサポート） |

### 商談での役割の原則
- **パートナーが主役**：顧客との関係はパートナーが主導する
- **自社は専門家として補足**：製品・技術・事例に関する質問に対応
- **価格・契約はパートナー経由**：自社から直接価格を提示しない
```

## ステップ4：ファイル出力

出力ファイル名の形式：
```
CHANNEL-STRATEGY-[パートナー名].md
```

例：`CHANNEL-STRATEGY-富士通株式会社.md`

ファイル内にパートナー向け提案書・エンドユーザー向け合同提案書・共同商談アジェンダをセクション分けして収録します。

## 重要なルール

- **パートナーファースト**：パートナーを飛び越えてエンドユーザーに直接営業しない。パートナーとの信頼関係が最優先
- **マージン・利益の考慮**：パートナーが適切な利益を得られる価格設計を提案に含める
- **資料の使い分け**：パートナー向け資料とエンドユーザー向け資料を明確に分離する。パートナー向けのマージン情報をエンドユーザーに見せない
- **パートナーの顧客情報の扱い**：パートナーが持つ顧客情報・案件情報を他のパートナーに共有しない
- **パートナーの育成**：長期的な協業のために、パートナーの営業担当者が自社製品を自走して売れるよう支援する

## 活用シーン

- 新規パートナー候補への協業提案
- 既存パートナーとの案件深耕・活性化
- パートナーとの合同商談前の準備
- パートナー向けトレーニング・キックオフ資料の土台
- パートナーが社内稟議を通すための支援資料
