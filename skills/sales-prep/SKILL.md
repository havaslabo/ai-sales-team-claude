# 商談準備ブリーフ

商談前に営業担当者が必要とするすべての情報を網羅した、包括的な商談準備ブリーフを生成します。企業調査、参加者情報、競合状況、そして実践的な準備内容を1つのアクション可能なドキュメントにまとめます。

## 呼び出し方法

```
/sales prep <url>
```

`<url>` は見込み企業のウェブサイトURLです。オプションとして、以下の情報も提供できます。
- 商談参加者の氏名
- 商談の日時
- 商談の目的・アジェンダ
- 提案する自社製品・サービス

## ステップ1：リサーチフェーズ

以下のリサーチタスクを実行してください。WebFetch を使用して各ソースからデータを収集します。準備時間を最小化するため、できる限り多くのフェッチを並行して実行してください。

### 1.1 企業調査

見込み企業のウェブサイトを取得し、以下を抽出します。

- **ホームページ**：企業説明、提供価値、主要メッセージ、ターゲット市場
- **会社概要ページ**：創業ストーリー、ミッション、チーム規模、オフィス所在地、企業理念
- **製品・サービスページ**：販売内容、価格モデル（公開されている場合）、主要機能
- **ブログ・ニュースページ**：最近の発表、コンテンツテーマ、思想リーダーシップのトピック
- **採用ページ**：募集中のポジション（成長領域、チームのギャップ、技術選択、予算配分の指標となる）
- **事例・お客様の声ページ**：顧客企業、強調している成果、対応業界

さらに、以下を検索します。
- 最近のプレスカバレッジやニュース（WebSearch で「[企業名] news [current year]」を使用）
- 最近の資金調達ラウンドや財務イベント（WebSearch で「[企業名] 資金調達 OR 買収 OR IPO」を使用）
- 企業のLinkedInページや企業公式SNSの活動状況（最近の投稿、フォロワー数、エンゲージメントパターン）

### 1.2 参加者調査

参加者の氏名が提供されている場合、各人物を調査します。

- **LinkedInプロフィール**：現在の役職、在籍期間、職歴、学歴、共通のつながり
- **最近のLinkedIn投稿や発信**：投稿テーマ、エンゲージするコンテンツ、職業的な関心事
- **カンファレンス登壇・ポッドキャスト**：公開での発言履歴は？どのようなトピックか？
- **寄稿記事や引用**：メディア出演や思想リーダーシップはあるか？

各参加者について、以下を含むプロフィールを作成します。
- 役職とシニアリティに基づくコミュニケーションスタイルの予測（細部重視、全体像重視、データ駆動型、関係重視型）
- ラポール形成のための個人的な関心事や接点（出身校・地域、投稿に記載の趣味、支持する活動）
- 役職に基づく推定優先事項（重視するKPI、懸念事項）
- **稟議・意思決定における役割**（担当者 / 課長 / 部長 / 役員など、承認フローを意識して把握する）

参加者名が提供されていない場合は、以下に基づいて推定参加者を特定します。
- 企業の役員紹介ページ
- 商談ステージと目的（初回訪問＝担当者・課長レベル、技術デモ＝情報システム部門、稟議段階＝経営層・財務部門）

### 1.3 競合状況

見込み企業が現在使用しているツール・サービス・ソリューションを調査します。

- **ウェブサイト技術**：既知のツールスクリプト、メタタグ、連携バッジ、「powered by」フッターを確認
- **求人情報**：採用ページや求人ボードでツール固有の要件を検索（例：求人説明文の「Salesforceの経験」）
- **パートナー・連携ページ**：テクノロジーパートナーの記載があるか？
- **ソーシャルでの言及**：企業や従業員が特定ツールについて投稿しているか？

特定した競合/現行ソリューションごとに以下を記録します。
- 見込み企業がそれを何に使用しているか
- そのソリューションに対する自社の優位点
- 避けるべきトピック（現行ベンダーを批判しない）

### 1.4 業界コンテキスト

見込み企業の業界を調査します。

- WebSearch を使用し、該当業界に関連する最新のトレンドや課題を2〜3件調査
- 業界に影響する規制変更、市場の変化、競合からの圧力を特定
- 自社ソリューションが対応できる業界固有の課題を探す

## ステップ2：商談ブリーフの作成

すべてのリサーチを以下のセクションにまとめます。各セクションは具体的でアクション可能な情報のみを記載し、一般的な内容は避けてください。

### セクション1：チートシート（クイックリファレンスカード）

ドキュメントの最上部に配置します。この商談に臨む前に覚えておくべき最重要事項5点を凝縮した1ページの参照シートです。

フォーマット：
```
## チートシート — [企業名] 商談

1. [この見込み企業について最も重要なこと — 1文で]
2. [ヒアリングすべき主要な課題・背景（初回は売り込まず、課題理解を目的とする）]
3. [最重要参加者とその稟議における役割・動機]
4. [現在使用しているソリューションと自社の最大の優位点]
5. [この商談で目指す唯一のアウトカム（例：課題ヒアリングの完了、次回アポの確保）]

**オープニング**：「[具体的でパーソナライズされた挨拶・関係構築の切り出し方]」
**初回ヒアリングの核心的な質問**：「[相手の状況・お困りごとを引き出す最重要質問]」
**避けるべき落とし穴**：「[この商談で言ってはいけないこと・やってはいけないこと]」
**名刺交換の準備**：[対面の場合は名刺を両手で。オンラインの場合は会社概要・自己紹介資料を事前に送付]
```

### セクション2：企業スナップショット

企業の全体像を伝える1段落（4〜6文）を作成します。以下を含めます。
- 事業内容と顧客対象
- 企業規模（従業員数、既知であれば売上、資金調達ステージ）
- 成長軌跡（採用シグナル、拡大の兆候、最近の発表）
- 営業会話に関連する主要なビジネスモデルの詳細

続いてクイックリファレンステーブルを掲載します。

| 項目 | 内容 |
|------|------|
| 企業名 | [名称] |
| ウェブサイト | [URL] |
| 業界 | [業界] |
| 設立年 | [年] |
| 従業員数 | [人数または推定] |
| 売上高 | [推定または範囲] |
| 資金調達 | [ステージと金額（既知の場合）] |
| 本社 | [所在地] |
| 主力製品 | [一覧] |
| ターゲット市場 | [販売対象] |

### セクション3：参加者プロフィール

参加者ごとに以下のプロフィールブロックを作成します。

```
### [フルネーム] — [役職]

**バックグラウンド**：[職歴、在籍期間、前職について2〜3文]
**最近の活動**：[最近の投稿、シェア、登壇内容]
**コミュニケーションスタイル**：[予測：細部重視 / 全体像重視 / データ駆動型 / 関係重視型]
**推定優先事項**：[役職に基づく関心事 — 具体的なKPI、目標、課題]
**稟議における役割**：[担当者 / 課長 / 部長 / 役員 / 情報収集者 / 推薦者 など]
**ラポールの接点**：[個人的な関心事、共通のつながり、出身校、地域、趣味]
**信頼を得るアプローチ**：[この人物に対する具体的なアプローチ — 強調すべき点、適切なトーン]
```

複数の参加者がいる場合は、以下も含めます。
- **稟議フローの推定**：担当者 → 課長 → 部長 → 役員など、決裁に関わる部署と承認ルートを想定する
- **社内推進者（チャンピオン）候補**：社内で話を前に進めてくれそうな人物はだれか？どうすれば推進者になってもらえるか？
- **商談内の力関係**：参加者間で注意すべきダイナミクスはあるか？

### セクション4：ビジネス状況

見込み企業の現在のビジネス状況を分析します。

- **現状**：現在の状況は？うまくいっていることは？課題は？
- **最近の変化**：経営陣の交代、製品ローンチ、方針転換、組織再編、または拡大はあるか？
- **成長軌跡**：成長中か、安定的か、縮小傾向か？その根拠は？
- **主要な課題**：リサーチに基づき、最も可能性の高い3〜5つの課題は何か？
- **対応できる機会**：自社ソリューションが直接解決できる課題はどれか？

課題は失敗としてではなく、機会として捉えて表現します。「問題」や「弱点」ではなく「改善余地」や「未開拓のポテンシャル」といった表現を使います。

### セクション5：競合コンテキスト

ステップ1.3の競合調査に基づきます。

- **現行ソリューション**：現在使用している各ツール/サービスと満足度の評価
- **切り替えの可能性**：現行ソリューションの具体的なトリガーやギャップ
- **言ってはいけないこと**：言及を避けるべき競合他社やトピックとその理由
- **自社の差別化**：この見込み企業の状況に特化した、自社を選ぶべき2〜3の最も説得力ある理由

### セクション6：トーキングポイント（5〜7点）

具体的でパーソナライズされた会話の切り出し方を5〜7点生成します。一般的なアイスブレイクではありません。各ポイントは見込み企業、その企業、または業界に関する具体的な内容に基づく必要があります。

**初回訪問の場合は「売り込み」ではなく「ヒアリング・関係構築」を目的とすること。** 自社製品の機能紹介から始めるのではなく、相手の状況・課題・理想の姿を理解することを優先してください。

フォーマット：
```
**トーキングポイント [#]**：[発言または質問]
**背景**：[なぜこれが関連するのか — どのリサーチが裏付けになるか]
**展開先**：[このトピックからどの話題・ヒアリング領域に発展できるか]
```

良いトーキングポイントの例：
- 最近の企業発表に言及し、その背景や狙いについて聞く
- 業界のトレンドに触れ、御社ではどのようにお感じか聞く
- 参加者が最近投稿・発言した内容に言及する
- 「現在、〇〇の領域でどのようなお困りごとがありますか？」と課題を引き出す
- 「理想の状態としてはどのようなイメージをお持ちですか？」と将来像を聞く

避けるべきトーキングポイントの例（生成不可）：
- 「ROIはどのくらいを想定していますか？」（初回では早すぎる）
- 「御社の予算はどのくらいですか？」（関係構築前にコストを聞くのは失礼）
- 「弊社のソリューションの機能をご説明します」（ヒアリング前の一方的な説明）

### セクション7：ディスカバリー質問（10問）

ラポール形成から課題の深掘りまで、順序立てた10のヒアリング質問を生成します。**日本のBtoB商談では、数値や予算より「お困りごと」「理想の状態」「背景・経緯」を先に深掘りすることが重要です。** 各質問について：

```
**Q[#]**：[質問]
**目的**：[どのような情報を引き出すか]
**想定される回答**：[リサーチに基づいて相手が言いそうなこと]
**フォローアップ**：[想定回答に対して次に聞くこと]
**注目するポイント**：[機会またはリスクを示すキーワードやシグナル]
```

質問の順序：
- 質問1〜2：ラポールとコンテキスト（「本日はお時間をいただきありがとうございます。まず、〇〇部門のご状況についてお聞かせいただけますか？」など）
- 質問3〜4：現状の把握（「現在、〇〇はどのように進めていらっしゃいますか？」「どのようなツールやプロセスをお使いですか？」）
- 質問5〜6：お困りごとの深掘り（「現在、〇〇に関してどのようなお困りごとがありますか？」「うまくいっていない点はどのあたりでしょうか？」）
- 質問7〜8：背景・影響（「その課題は、どのようなきっかけで顕在化したのでしょうか？」「その影響は他の部門にも出ていますか？」）
- 質問9〜10：理想の状態と稟議プロセス（「理想の状態としてはどのようなイメージをお持ちですか？」「このようなご検討は、どのような流れで進んでいくことが多いですか？稟議に関わる部署はどちらでしょうか？」）

### セクション8：想定される反論（5つ）

企業規模、業界、現行ソリューション、商談ステージに基づき、この見込み企業が最も提起しそうな懸念・反論を5つ特定します。

各反論に対し、相手の立場を尊重した準備された回答を提供します。

```
**反論 [#]**：「[相手が言いそうな具体的な言葉]」
**真の懸念**：[反論の裏にある本当の懸念]
**回答**：
「[懸念に共感しながら、押しつけがましくなく背景を補足する。例：
「おっしゃる点はよくわかります。同様のご状況だった〇〇様のケースでは、
[具体的な取り組みや経緯]という形で進めることができました。
御社のご状況に合わせた形でご検討いただける方法があればと思っております。」]」
**証拠**：[使用すべき具体的なケーススタディ、実績、または参考事例]
```

### セクション9：成功指標

具体的で測定可能なアウトカムを定義します。

- **最低限の成功**：この商談を意義あるものにする最低限のアウトカム
- **目標とする成功**：目指すべき現実的な最善のアウトカム
- **理想の成功**：すべてが完璧にいった場合の理想のアウトカム

各成功レベルについて、以下を明記します。
- 見込み企業が言うこと、または同意すること
- 確立される具体的なネクストステップ
- 持ち帰る情報

### セクション10：競合の地雷

この商談で意図的に避けるべきトピック・競合・話題の一覧：

```
**地雷 [#]**：[避けるべきトピック]
**理由**：[例：競合と密接な関係がある、センシティブなトピック、会話が脱線する可能性]
**万が一話題になった場合**：[見込み企業が持ち出してきた場合の対応方法]
```

### セクション11：提案するネクストステップと社内推進者への対応

商談終了時に提案する具体的で期限を持った2〜3のネクストステップを積極性の順に準備します。

1. **大胆な提案**：最も野心的なネクストステップ（例：より広いチームへのデモ、上位決裁者との面談設定）
2. **標準的な提案**：合理的なネクストステップ（例：課題整理の資料送付 → フォローアップ商談、参考事例の紹介）
3. **最低限の提案**：フォールバック（例：翌週のフォローアップコール、追加情報の送付）

各ステップについて、以下を提供します。
- 提案時の具体的な言葉（押しつけにならないよう「〜はいかがでしょうか」「ご都合よろしければ〜」を活用）
- 具体的な日時の提案（例：「来週木曜の午後はいかがでしょうか？」）
- 見込み企業がネクストステップを断った場合の対応

**社内推進者（チャンピオン）になってもらうためのアクション：**
- 今回の商談でチャンピオン候補になりそうな人物は誰か
- 稟議を社内で進めてもらうために提供できる資料・情報（経営層向け要約、費用対効果の整理、他社事例など）
- チャンピオンが社内で説明しやすくなるためのサポート方法

## ステップ3：アジェンダテンプレート

商談の種類と時間に基づいた推奨商談構成を生成します。

**日本の商談文化の注意点：**
- 初回訪問は「自社の説明・ヒアリング」が主目的。製品説明より先に関係構築と課題把握を行う
- 対面の場合は冒頭に名刺交換の時間を確保する（オンラインの場合は参加者の確認と自己紹介を丁寧に行う）
- 商談の最後は「本日はお時間をいただきありがとうございました」と感謝を伝えてからネクストステップを提案する

### 30分商談
| 時間 | アクティビティ | 備考 |
|------|--------------|------|
| 0:00-3:00 | 挨拶・名刺交換・自己紹介 | 対面：名刺を両手で。オンライン：参加者確認・会社概要の共有 |
| 3:00-12:00 | ヒアリング（課題・現状の把握） | 優先する上位4つの質問。売り込まず、聴くことに集中する |
| 12:00-22:00 | 御社の課題解決という視点での価値紹介 | 相手の課題に引き寄せた提案。機能説明より課題解決の文脈で話す |
| 22:00-27:00 | Q&A / 懸念事項の確認 | 相手の懸念を丁寧に引き出す |
| 27:00-30:00 | 御礼・ネクストステップの提案 | 具体的な日時を含む提案。「ご検討いただければ幸いです」の姿勢で |

### 60分商談
| 時間 | アクティビティ | 備考 |
|------|--------------|------|
| 0:00-5:00 | 挨拶・名刺交換・自己紹介・会社概要の共有 | 対面：名刺交換。オンライン：会社概要・自己紹介資料の確認 |
| 5:00-25:00 | ヒアリング深掘り（お困りごと・理想の状態・背景） | 10の質問、優先順に。数値より課題・背景の深掘りを優先する |
| 25:00-45:00 | 御社の課題解決に向けた提案・事例紹介 | 相手の課題に合わせてカスタマイズ。プロダクト機能より課題解決から始める |
| 45:00-53:00 | ディスカッション・懸念事項の確認 | 懸念への対応、同様の成功事例の提示 |
| 53:00-57:00 | 稟議プロセスの確認・追加情報の確認 | 「稟議はどのように進まれますか？」「他にご確認される方はいらっしゃいますか？」 |
| 57:00-60:00 | 御礼・ネクストステップの合意 | 具体的な日時を含む提案。感謝を伝えてから締める |

## ルールと制約

1. **すべてはこの見込み企業に固有でなければなりません。** 一般的なアドバイスは不可です。すべてのトーキングポイント・質問・反論・推奨事項は、リサーチから得た具体的な詳細を参照する必要があります。
2. **証拠に基づく主張のみ。** すべての事実的な主張についてソースを明記します（ウェブページ、ニュース記事、LinkedIn投稿）。推測の場合は必ず推測である旨を明記します。
3. **見込み企業の知性を尊重してください。** 操作的な戦術、心理的圧力テクニックは含めません。これはプロフェッショナルなビジネス会話のための準備です。
4. **網羅性より実用性を優先。** 営業担当者がチートシートを60秒で読んで自信を持って商談に臨めることが重要です。深い内容は補足セクションに収めます。
5. **参加者名が提供されていない場合**も、商談の種類と企業規模に基づく推定参加者を使って参加者プロフィールセクションを作成します。「推定参加者」とラベルを付け、確信度を記載します。
6. **過去の分析ファイルが存在する場合**（PROSPECT-ANALYSIS.md、COMPANY-RESEARCH.md、LEAD-QUALIFICATION.md、COMPETITIVE-INTEL.md、DECISION-MAKERS.md）、それらを読み込んで知見を組み込みます。既に分析済みの内容は再調査しません。
7. **情報の鮮度に注意。** 最近変更された可能性のある情報（経営陣、資金調達、製品ローンチ）はWebSearchで確認します。各ソースの日付を記録します。
8. **チートシートが最重要セクションです。** 営業担当者が他を読まなくても、チートシートだけで何も準備しないより格段に準備が整った状態で臨めるようにします。
9. **ヒアリング質問は純粋な好奇心と相手への関心から生まれるものでなければなりません。** 相手の状況・課題・理想を本当に理解しようとする質問であるべきです。予め決まった結論に誘導するための質問ではありません。
10. **競合への言及は事実に基づく必要があります。** 競合インテリジェンスを捏造しません。競合の弱点を検証できない場合は「一般的に報告されている」と記載するか、省略します。
11. **初回訪問は関係構築・ヒアリングが目的であることを忘れないでください。** 日本のBtoB営業では、初回から製品を売り込むことはむしろ逆効果になる場合があります。まず相手を理解し、信頼関係を築くことを最優先にしてください。
12. **稟議プロセスを意識したアプローチを組み込んでください。** 日本のBtoB企業では、担当者→課長→部長→役員と複数の承認者が関わることが多いです。チャンピオン（社内推進者）を見つけ、稟議を支援するための情報・資料提供を戦略に含めてください。
